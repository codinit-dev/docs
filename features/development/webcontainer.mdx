CodinIT’s **WebContainer** provides a complete Node.js development environment running entirely in your browser.  
Built on [StackBlitz’s WebContainer API](https://webcontainers.io), it eliminates the need for local setup while providing full Node.js capabilities.

---

## Core Features

<ParamField type="runtime">
Complete Node.js environment including npm, yarn, and all standard Node.js APIs running in your browser.

- **Process Management**: Spawn and manage multiple Node.js processes  
- **File System**: Full read/write access to project files  
- **Network**: HTTP servers, WebSocket connections, and API calls  
- **Environment Variables**: Full Node.js env variable support
</ParamField>

<ParamField type="package">
Native support for npm and yarn package management.

- **Dependency Installation**: Install packages from npm registry  
- **Script Execution**: Run npm scripts and build processes  
- **Package Resolution**: Automatic dependency resolution and caching  
- **Lock File Support**: Compatible with `package-lock.json` and `yarn.lock`
</ParamField>

---

### Preview System

<ParamField type="preview">
Real-time preview of your applications with hot reload support.

- **Instant Updates**: See changes reflected immediately  
- **Error Overlay**: Visual error indicators with stack traces  
- **Cross-Origin Support**: Handle COEP requirements for modern web features  
- **Port Management**: Automatic port allocation and management
</ParamField>

<ParamField type="errors">
Comprehensive error forwarding from preview iframes.

- **Uncaught Exceptions**: Capture and display JavaScript errors  
- **Promise Rejections**: Handle unhandled promise rejections  
- **Stack Traces**: Detailed stack traces with source mapping  
- **Error Context**: File path, line numbers, and execution context
</ParamField>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Container Won’t Start">
    **Error**: WebContainer fails to initialize  
    **Fixes**:
    - Check browser compatibility  
    - Clear cache and reload  
    - Disable conflicting extensions  
    - Ensure stable connection
  </Accordion>

  <Accordion title="Preview Not Loading">
    **Error**: Blank preview or runtime errors  
    **Fixes**:
    - Check JavaScript console  
    - Verify dependencies  
    - Ensure dev server runs correctly  
    - Check for port conflicts
  </Accordion>

  <Accordion title="Performance Issues">
    **Error**: Slow or laggy experience  
    **Fixes**:
    - Close unused tabs  
    - Restart WebContainer  
    - Check for memory leaks  
    - Optimize bundle size
  </Accordion>

  <Accordion title="Network Problems">
    **Error**: Package installs or API calls fail  
    **Fixes**:
    - Check internet stability  
    - Verify npm registry access  
    - Clear WebContainer cache  
    - Try alternate npm registry
  </Accordion>
</AccordionGroup>
